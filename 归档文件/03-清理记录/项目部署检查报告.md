### 1. 创建环境变量模板文件

**重要**: 由于系统限制，`.env.example` 文件需要手动创建。

请创建 `.env.example` 文件，内容如下：

```env
# Django配置
# 生产环境请务必修改SECRET_KEY为随机字符串
SECRET_KEY=django-insecure-=sg+j-mtq#xt*0x3^*4vdu9vzn=qrl^6e&7u@7qav$h8b&op8(
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1

# 数据库配置
DB_NAME=educloud
DB_USER=root
DB_PASSWORD=123456
DB_HOST=127.0.0.1
DB_PORT=3306

# 学期配置（可选）
SEMESTER_START_DATE=2024-09-01
SEMESTER_TOTAL_WEEKS=20
```

### 2. 部署前准备

1. **创建 `.env` 文件**
   ```bash
   # Windows
   copy .env.example .env
   
   # Linux/Mac
   cp .env.example .env
   ```

2. **配置 `.env` 文件**
   - 修改 `SECRET_KEY` 为随机值
   - 配置数据库连接信息
   - 生产环境设置 `DEBUG=False`

3. **准备数据库**
   - 创建MySQL数据库
   - 配置数据库用户和权限

## 📋 部署步骤

### 快速部署（推荐）

**Windows:**
```bash

copy .env.example .env

# 2. 编辑.env文件配置数据库等信息

# 3. 运行部署脚本
deploy.bat
```

**Linux/Mac:**
```bash
# 1. 添加执行权限
chmod +x deploy.sh

# 2. 创建.env文件（如果不存在）
cp .env.example .env

# 3. 编辑.env文件配置数据库等信息

# 4. 运行部署脚本
./deploy.sh
```

## 🔒 安全建议

1. **生产环境必须设置**
   - `DEBUG=False`
   - `SECRET_KEY` 改为随机字符串
   - `ALLOWED_HOSTS` 限制为实际域名

2. **数据库安全**
   - 使用强密码
   - 限制数据库用户权限
   - 定期备份数据库

3. **文件权限**
   - `.env` 文件权限设为 600（Linux/Mac）
   - 不要将 `.env` 提交到版本控制
