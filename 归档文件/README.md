# 归档文件目录

本目录用于存放项目中不需要的文件，按功能分类整理。

## 📁 目录结构

```
归档文件/
├── 01-临时报告/          # 临时生成的报告文件
├── 02-部署配置/          # 部署相关配置文件（如不需要可归档）
├── 03-清理记录/          # 项目清理相关文档
├── 04-一次性脚本/        # 已完成任务的一次性脚本
└── README.md            # 本说明文件
```

## 📋 文件分类说明

### 01-临时报告/
存放临时生成的报告文件，这些文件通常由工具自动生成，可以随时删除。

**文件说明：**
- `migration_file_report.json` - 数据库迁移文件完整性校验报告
  - 由 `accounts/management/commands/verify_files.py` 命令生成
  - 用于验证文件迁移的完整性
  - 可以删除，需要时可重新生成

### 02-部署配置/
存放部署相关的配置文件，如果项目不需要部署到特定平台，可以归档这些文件。

**文件说明：**
- `vercel.json` - Vercel平台部署配置文件
  - 包含URL重写规则
  - 如果不需要部署到Vercel，可以归档
- `.vercelignore` - Vercel部署忽略规则文件
  - 指定Vercel部署时忽略的文件
  - 如果不需要部署到Vercel，可以归档

### 03-清理记录/
存放项目清理相关的文档记录。

**文件说明：**
- `CLEANUP_SUMMARY.md` - 项目清理总结文档
  - 记录了项目清理过程中删除的文件和原因
  - 作为历史记录保留

### 04-一次性脚本/
存放已完成任务的一次性脚本，这些脚本已完成其使命，不再需要。

**文件说明：**
- `organization/management/commands/update_class_ids.py` - 更新班级ID脚本
  - 用于批量更新班级ID格式
  - 任务已完成，可以归档

## ⚠️ 注意事项

1. **删除前确认**：归档文件可以删除，但建议先确认不再需要
2. **Git历史**：如果文件已提交到Git，删除后仍可从历史记录恢复
3. **重新生成**：某些报告文件可以重新生成，删除不影响功能
4. **部署配置**：如果将来需要部署，可以从归档目录恢复配置文件
