# 归档文件说明

## 📦 归档目的

本目录用于整理项目中不需要的文件，保持项目根目录整洁，便于管理和维护。

## 📁 分类说明

### 01-临时报告/
**用途**：存放临时生成的报告文件

**特点**：
- 由工具自动生成
- 可以随时删除
- 需要时可重新生成

**文件**：
- `migration_file_report.json` - 迁移文件完整性校验报告

### 02-部署配置/
**用途**：存放部署相关的配置文件

**特点**：
- 特定平台的部署配置
- 如果不需要部署可以归档
- 需要部署时可恢复

**文件**：
- `vercel.json` - Vercel部署配置
- `.vercelignore` - Vercel忽略规则

### 03-清理记录/
**用途**：存放项目清理相关的文档记录

**特点**：
- 历史记录文档
- 记录清理过程
- 可作为参考

**文件**：
- `CLEANUP_SUMMARY.md` - 项目清理总结

### 04-一次性脚本/
**用途**：存放已完成任务的一次性脚本

**特点**：
- 任务已完成
- 不再需要运行
- 可作为参考

**文件**：
- `update_class_ids.py` - 更新班级ID脚本

## 🔄 恢复文件

如果需要恢复某个文件到原位置：

### 恢复临时报告
```bash
# 不需要恢复，可以重新生成
python manage.py verify_files --src <源目录> --dst <目标目录> --output migration_file_report.json
```

### 恢复部署配置
```bash
# 恢复Vercel配置
cp 归档文件/02-部署配置/vercel.json .
cp 归档文件/02-部署配置/.vercelignore .
```

### 恢复清理记录
```bash
# 恢复清理总结文档
cp 归档文件/03-清理记录/CLEANUP_SUMMARY.md .
```

### 恢复一次性脚本
```bash
# 恢复更新班级ID脚本
cp 归档文件/04-一次性脚本/update_class_ids.py organization/management/commands/
```

## 🗑️ 删除归档文件

归档文件可以安全删除，但建议：
1. 确认不再需要这些文件
2. 备份重要文件
3. 了解文件用途后再删除

## 📝 添加新文件到归档

如果发现新的不需要的文件，可以：

1. **确定分类**：根据文件类型选择对应的分类目录
2. **移动文件**：将文件移动到对应目录
3. **更新说明**：在对应目录的README.md中添加文件说明
4. **更新总览**：更新本目录的README.md

## ⚠️ 注意事项

1. **Git跟踪**：归档文件会被Git跟踪，删除后仍可从历史恢复
2. **备份重要文件**：删除前建议备份重要文件
3. **恢复方法**：了解如何恢复文件，以备不时之需
4. **定期清理**：建议定期检查归档文件，删除确实不需要的文件
