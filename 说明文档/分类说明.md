## 📋 分类原则

1. **按文档用途分类**
   - 使用指南 → `01-使用指南`
   - 技术文档 → `02-技术文档`
   - 修复记录 → `03-修复记录`

2. **按功能模块分类**
   - 3D可视化相关 → `04-3D可视化`
   - 课程管理相关 → `05-课程管理`

3. **优先级规则**
   - 如果文档既是使用指南又是技术文档，优先放在使用指南
   - 如果文档涉及多个分类，放在最主要功能模块下
   - 修复类文档统一放在修复记录目录

## 📝 添加新文档指南

### 如何确定新文档的分类？

1. **使用指南类**: 如果文档是教用户如何使用功能
   → 放在 `01-使用指南`

2. **技术文档类**: 如果文档是给开发人员看的技术细节
   → 放在 `02-技术文档`

3. **修复记录类**: 如果文档是记录问题修复过程
   → 放在 `03-修复记录`

4. **功能模块类**: 如果文档是某个特定功能的说明
   → 放在对应的功能模块目录（如 `05-课程管理`）
